What's PhoneGap

What is PhoneGap? PhoneGap is powered by Apache Cordova and provides commerical services (some are for free) on top of Cordova.

For example, the PhoneGap team released the The PhoneGap Developer. The idea is you can install Node.js and PhoneGap on your computer, download the PhoneGap Developer App on your phone, and start developing apps without installing native SDK's or worrying about signing code. You can't distribute these apps but it works well for development.

PhoneGap offers a cloud build service to produce distributable apps.

As Cordova is the foundation it definitely made sense to get in touch with the PhoneGap community to learn about common topics and latest trends.

Hands-on Bluetooth Low Energy

This talk was all about using the Bluetooth Low Energy (BLE) Central cordova plugin written by Don Coleman to read and write data from Texas Instruments' SensorTag Device.

We all received the device above for us to play with basically we were able to do the following tasks:

  • Scan all the BLE compatible devices
  • Connect to the BLE device and read the available services
  • Change the characteristic configuration so we can read or write data to it.
  • Read sensor data such as temperature, button press, magnetic sensor.
  • Control the LED on the SensorTag by using javascript alone.

App Privacy and Security for PhoneGap Developers

Speaker: Tommy Williams (@theRealDevgeeks)

Security as a complex field and therefore the pesentation can only be considered as an overview session

Pretty much the same risks / attacks as for browser apps

  • Cross Site Scripting (XSS)
  • Cross Site Request Forgery (CSRF)
  • SQL Injection which can easily be tested with SQLMap
  • Man in the Middle (MitM) attachk
  • ...

Cordova-specific security mechanisms:

  • Whitelist Plugin and Content Security Policy (CSP) since Cordova 5
  • SSL Certificate Checker Plugin for approximate certificate pinning or Cordova-HTTP plugin for true certificate pinning (for some platforms)

Hands on: Create advanced apps with the Crosswalk runtime

Speaker: Alexis Menard

This isn't really a hands-on talk because we basically just watch and listen to the speaker.

In this talk, the speaker gave us an overview of what Crosswalk is and also presented an overview of the technical architecture of Crosswalk.

In a nutshell, Crosswalk brings the latest Chromium to Android devices running from Android 4.0 up to provide bug fixes and performance improvements even for older devices. This is really great for developers targeting devices from Android 4.0 and up. The only consequence is that your application binary size increases significantly.

Finding the sweet spot - blending the best of native and web

Speaker: Shawn Jansepar (@shawnjan8)

Why to choose between web or native? Experience matters and not technology.

Experience from Mobify projects: building some components natively for smooth and natural look and feel (especially navigation items) or high-performance critical elements and the rest are web view. P.S.: Amazon is using this concept as well.

Specific handling for web parts by detecting in which context the web content is running. For example, Mobify app detects the "app" in 3 places: app class in CSS, isRunningInApp function in JS and appending info to user agent for server-side handling

Options:

  • Embedded PhoneGap (to build a native hbyrid instead of a web hybrid)
  • Astro from Mobify

PhoneGap Applications with Ember: Lessons learned

Speaker: Alexis (@alexblom) from Isle of Code

Lessons Learned and best tips & tricks about Ember.js but also in general. Best of as follows:

  • ember-cli-cordova
  • touch events: use hammerjs
  • webview: use crosswalk
  • JS: nullify your variables to ensure garbage collection; name your closures to make profiling in Chrome Debugger console easier; cut down reflow by use of CSS Transforms; avoid unnecessary div nesting
  • -webkit-tap-highlight-color:rba(0,0,0,0): don't highlight links after they were clicked
  • Liquid fire library to handle routing animations (http://ef4.github.io/liquid-fire; specific to Ember.js; checkout othe libs)
  • Don't use jQuery Animations on mobile!!! Use rather velocity.js

Conclusion
